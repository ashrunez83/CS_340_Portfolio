To write maintainable, readable, and adaptable programs, I focus on modular design, clear naming conventions, and separating responsibilities within the code. In Project One, I developed a reusable CRUD Python module that handled database interactions independently from the dashboard interface. This separation allowed the dashboard in Project Two to communicate with the database without embedding raw queries directly into the UI logic. The advantage of this approach was flexibility: if the database structure changed or new queries were needed, I could modify the CRUD module without restructuring the entire dashboard application. In the future, I could reuse this CRUD module as a foundation for other database-driven applications, such as analytics dashboards, reporting tools, or administrative interfaces that require consistent data operations.

When approaching the database and dashboard requirements for Grazioso Salvare, I treated the project as a structured problem-solving process. I began by identifying user requirements, translating them into data structures, and designing queries that would efficiently retrieve and display the requested information. Unlike some earlier assignments that focused primarily on syntax or isolated functions, this project required end-to-end system thinking—connecting a database, business logic, and user interface into one cohesive system. In the future, I would continue using a requirements-first approach: analyzing client needs, modeling the data appropriately, creating modular components, and testing iteratively to ensure reliability and scalability.
